## Key design decisions (grounded in the recon)

1. **Static singletons, not Hilt, for the node layer.** `RelaisEngine`, `RelaisConfig`,
   `RelaisNodeService` are all `object`s/statics accessed with a `Context`; the node's HTTP/service code
   never uses Hilt injection (Hilt only wires the gallery's DataStore/repository layer via
   `di/AppModule.kt`). So `RelaisDatabase` and `RelaisInference` follow the **`object` + `get(context)`**
   idiom (like `RelaisConfig`), NOT `@Provides`. This keeps the node layer uniform and avoids dragging
   Hilt into `RelaisHttpServer`/workers. *(A Hilt `@Provides RelaisDatabase` could be added later for
   the gallery side without conflict.)*
2. **`RelaisNodeService.start/stop` already exist** (lines 148–160) and set `RelaisConfig.setShouldRun`.
   `RelaisNodeController` is therefore a thin **read-model** (`state()`) over existing signals
   (`RelaisConfig.shouldRun` = operator intent, `RelaisEngine.isReady` = engine resident,
   `RelaisEngine.startupInProgress`, and a new `lastInitFailed`) plus pass-through `start()/stop()`.
3. **Room DB must compile with ≥1 entity** → core-infra ships a `SchemaMeta` marker entity at
   **version 1** and establishes the migration pattern; #4/#5/#14 each add entities, bump the version,
   and add a `Migration`. `exportSchema = true` (committed schema JSON) enables migration tests.

## Implementation sketch

### `RelaisInference` — the in-process facade
```kotlin
package cc.grepon.relais.core

import android.content.Context
import cc.grepon.relais.RelaisEngine
import cc.grepon.relais.RelaisRequest
import kotlinx.coroutines.channels.awaitClose
import kotlinx.coroutines.flow.Flow
import kotlinx.coroutines.flow.callbackFlow
import kotlinx.coroutines.flow.flow

/**
 * In-process one-shot inference for NON-HTTP callers (tiles, widgets, share, NFC, triage, Tasker).
 * Wraps [RelaisEngine.generate]. CRITICAL: this is a *consumer* of an already-resident engine — it
 * does NOT cold-start / provision a multi-GB model from a UI tap. If the engine isn't ready it fails
 * fast (callers surface "node not running"), so a home-screen tap can never trigger a model download
 * with no foreground service behind it.
 */
object RelaisInference {

  class NodeNotReadyException : IllegalStateException("Relais node is not running")

  /** True iff a resident engine can serve right now. */
  fun isReady(): Boolean = RelaisEngine.isReady

  /**
   * Streams visible answer deltas. Reasoning (thinking) deltas are dropped here (callers that want
   * them use the HTTP path / a future overload). Throws [NodeNotReadyException] if the engine is down.
   * Runs the blocking engine call on the collector's dispatcher — call from Dispatchers.IO / a Worker.
   */
  fun complete(
    context: Context,
    prompt: String,
    system: String? = null,
    images: List<android.net.Uri> = emptyList(),
  ): Flow<String> = callbackFlow {
    if (!RelaisEngine.isReady) throw NodeNotReadyException()
    val req = RelaisRequest(
      text = prompt,
      systemPrompt = system,
      imagePng = images.firstOrNull()?.let { readPngBytes(context, it) }, // single image v1
    )
    // Engine is blocking + single-flight; emit tokens via onToken, close on return.
    val result = RelaisEngine.generate(
      context = context,
      request = req,
      onToken = { delta -> trySend(delta) },
    )
    if (RelaisEngine.isReady) { /* result.text already streamed */ }
    close()
    awaitClose { }
  }

  /** Convenience: full text (collects the Flow). */
  suspend fun completeText(context: Context, prompt: String, system: String? = null): String {
    val sb = StringBuilder()
    complete(context, prompt, system).collect { sb.append(it) }
    return sb.toString()
  }

  private fun readPngBytes(context: Context, uri: android.net.Uri): ByteArray? =
    runCatching { context.contentResolver.openInputStream(uri)?.use { it.readBytes() } }.getOrNull()
}
```

### `NodeState` + `RelaisNodeController`
```kotlin
// NodeState.kt — pure mapper, JVM-testable (mirrors the tile's needs).
package cc.grepon.relais.core

enum class NodeState { OFF, STARTING, LIVE, HOT, ERROR }

/** Maps the node's raw signals to a single UI state. thermalStatus is ThermalGovernor.statusValue (0..6). */
fun computeNodeState(
  shouldRun: Boolean, ready: Boolean, startupInProgress: Boolean,
  lastInitFailed: Boolean, thermalStatus: Int,
): NodeState = when {
  ready && thermalStatus >= 4 -> NodeState.HOT          // THERMAL_STATUS_SEVERE+
  ready                       -> NodeState.LIVE
  shouldRun && lastInitFailed -> NodeState.ERROR        // asked-on but init threw
  shouldRun || startupInProgress -> NodeState.STARTING
  else                        -> NodeState.OFF
}
```
```kotlin
// RelaisNodeController.kt — thin read-model + lifecycle delegate over EXISTING statics.
package cc.grepon.relais.core

import android.content.Context
import cc.grepon.relais.RelaisConfig
import cc.grepon.relais.RelaisEngine
import cc.grepon.relais.RelaisNodeService
import cc.grepon.relais.ThermalGovernor

object RelaisNodeController {
  fun isRunning(context: Context): Boolean = RelaisConfig.shouldRun(context)  // operator intent latch
  fun state(context: Context): NodeState = computeNodeState(
    shouldRun = RelaisConfig.shouldRun(context),
    ready = RelaisEngine.isReady,
    startupInProgress = RelaisEngine.startupInProgress,
    lastInitFailed = RelaisEngine.lastInitFailed,
    thermalStatus = ThermalGovernor.statusValue,
  )
  fun start(context: Context) = RelaisNodeService.start(context)  // existing static
  fun stop(context: Context) = RelaisNodeService.stop(context)    // existing static
}
```

### `RelaisDatabase` (Room) — the migration root
```kotlin

@Entity(tableName = "schema_meta")

```
```kotlin
// RelaisDatabase.kt — static singleton, matching the RelaisConfig/RelaisEngine idiom.
package cc.grepon.relais.data
import android.content.Context
import androidx.room.*
import androidx.room.migration.Migration

@Database(entities = [SchemaMeta::class], version = 1, exportSchema = true)
abstract class RelaisDatabase : RoomDatabase() {
  abstract fun schemaMetaDao(): SchemaMetaDao
  // #4 adds ragDao(); #5 adds sessionDao(); #14 adds batchJobDao(); each bumps version + adds a Migration.

  companion object {
    private const val DB_NAME = "relais.db"
    @Volatile private var instance: RelaisDatabase? = null

    /** Process-wide singleton (single process — see backlog §3). */
    fun get(context: Context): RelaisDatabase =
      instance ?: synchronized(this) {
        instance ?: Room.databaseBuilder(context.applicationContext, RelaisDatabase::class.java, DB_NAME)
          .addMigrations(*MIGRATIONS)
          // No destructive fallback: on-device data (RAG corpus, sessions, batch jobs) must survive
          // upgrades — consumers MUST supply real migrations when they bump the version.
          .build()
          .also { instance = it }
      }

    /** Consumers append their Migration(n, n+1) here when they add tables. Empty at v1. */
    val MIGRATIONS: Array<Migration> = arrayOf()

    @VisibleForTesting fun resetForTest() { instance = null }
  }
}
```

## Security considerations
- **No new attack surface.** Pure plumbing: no new routes, services, exported components, or
  permissions. `RelaisInference` runs in-process and is reachable only by in-app callers (it does not
  bypass the HTTP bearer gate because it isn't on the network path).
- **`RelaisInference` cannot cold-start the engine** — `isReady` guard prevents a UI tap from
  triggering a multi-GB provision/download with no foreground service (would be OOM-killed). This
  guard is the security/robustness crux that the tile/widget plans rely on.
- **Room DB is app-private** (`getDatabasePath`, not world-readable). It is created empty here; the
  privacy posture of what gets stored is owned by the consumers (#4 corpus, #5 history) and documented
  there. **No destructive migration fallback** — consumer data must survive upgrades.
- `RelaisEmbedderProvider.get()` returning null is the single source of truth both `/v1/embeddings`
  (→501) and RAG (→disabled) read; no partial/garbage-vector path can exist before #6 lands.

## Devil's Advocate
1. **Is `RelaisNodeController` even worth it, given `RelaisNodeService.start/stop` + `RelaisConfig.shouldRun` already exist?** **[NOTED]** — yes, for the unified `state()` read-model (#2/#3 both need OFF/STARTING/LIVE/HOT/ERROR); without it each surface re-derives state and they drift. The lifecycle methods are thin delegates by design.
2. **`RelaisInference.complete` via `callbackFlow` + a blocking engine call — back-pressure / thread.** **[BLOCKER]** — `trySend` can drop on a full buffer; the engine call blocks the collector. Resolution in delta: use a buffered channel (`callbackFlow` default + `buffered()`), document "collect on Dispatchers.IO", and since the engine is single-flight + blocking the producer is naturally paced.
3. **Empty Room DB with a marker entity is slightly artificial.** **[NOTED]** — but a zero-entity `@Database` won't compile, and a real migration root (v1 + exported schema + `MIGRATIONS`) is exactly what makes #4/#5/#14 cheap and safe. The `SchemaMeta` row is also a useful "DB created at / by version" record.
4. **Room version vs the catalog's KSP line could break the build.** **[BLOCKER]** — pinned mitigation: a `kspDebugKotlin` smoke test in the PR; pin Room to whatever matches the existing KSP if 2.7.1 conflicts.
5. **`RelaisInference` drops reasoning deltas + handles only one image.** **[NOTED]** — intended v1 scope for UI surfaces (tiles/widgets want the answer, not the chain-of-thought); an overload can add reasoning/multi-image later. Documented in KDoc.
6. **Static singletons vs the existing Hilt module — inconsistency?** **[NOTED]** — deliberate: the *node* layer is static-with-Context throughout; Hilt is the *gallery* layer. Matching the node idiom keeps `RelaisHttpServer`/workers Hilt-free. A Hilt `@Provides RelaisDatabase` can wrap the singleton later for gallery-side injection.

## Revised plan delta (address BLOCKERs)
- **#2:** `RelaisInference.complete` uses `callbackFlow` with the default buffered channel; KDoc mandates
  collection on `Dispatchers.IO`; because `RelaisEngine.generate` is blocking + single-flight, the
  producer is paced by the collector and `trySend` won't realistically overflow — but add a
  `.buffer(Channel.UNLIMITED)`-equivalent only if a consumer reports drops (YAGNI otherwise). Provide
  the `completeText` suspend convenience as the primary API for non-streaming callers (tile/Tasker/
  triage) so most callers never touch the Flow.
- **#4:** Add a mandatory `./gradlew :app:kspDebugKotlin` step to the PR's CI/verify list; if Room 2.7.1
  fails against the catalog KSP, pin Room to the matching line (documented as the single build risk).
- Keep `lastInitFailed` a one-line `@Volatile` on `RelaisEngine` set only from the existing
  `RelaisNodeService` init try/catch — no change to `generate`'s hot path.
